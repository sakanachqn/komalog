# コマログ

駒を集めて盤面に配置し、自動戦闘で全3幕を攻略するHTML5ローグライト。
分岐マップを進み、シナジー・アイテム・レリックを組み合わせて最後に待つ魔王の討伐を目指します。

## 主な特徴

- 全3幕の分岐マップ（戦闘、エリート、ショップ、休憩、イベント、ボス）
- コスト1〜5のユニット92体と、全22系統のシナジー
- 同じユニット3体で星アップ。★3完成後は売却するまでショップに再登場しない
- 通常アイテム8種と、2つの素材から作る合成アイテム36種
- 通常レリック39種、古代レリック40種
- 全シナジーに対応する専用古代レリック
- ユニット、シナジー、アイテム、レリック、古代レリックを確認できる図鑑
- 記憶の欠片を使う恒久解放「記憶の祭壇」
- 31個の実績と、全実績解除時の特別なタイトル画面
- 段位0〜20のアセンション
- 戦闘終了後のユニット別バトルレポート
- タイトルから遊べる実戦チュートリアル
- タイトル、各ステージで切り替わるループBGM
- SE・BGM音量、画面揺れ、演出軽減などの設定
- ローカル環境限定の図鑑・実績確認用デバッグ機能

## 遊び方

1. タイトル画面で段位とスターターを選び、ランを開始します。
2. マップ上の進行先を選び、戦闘やショップ、イベントを巡ります。
3. ユニットを購入・獲得し、盤面とベンチを整えます。
4. 同じシナジーを持つユニットを規定数配置して部隊を強化します。
5. 各幕のボスを倒し、第3幕の最終ボスを撃破するとクリアです。

プレイヤーHPが0になるとラン終了です。進行度に応じて「記憶の欠片」を獲得し、次回以降のランで使えるユニットや機能を解放できます。

## 操作

- ユニットの配置、移動、入れ替え：ドラッグ＆ドロップ
- ベンチ内の並び替え：ドラッグ＆ドロップ
- ショップのミニ盤面：盤面とベンチ間をドラッグ＆ドロップ
- アイテム装備・合成：アイテムを対象ユニットまたは合成枠へドラッグ
- ユニット、アイテム、シナジー、レリックの詳細：対象へカーソルを合わせる
- 戦闘速度：等速、2倍、3倍（設定は次の戦闘にも保持）

配置上限に満たない状態で戦闘を開始すると、ベンチ左端から不足分が自動配置されます。

## ゲームシステム

### ユニットとシナジー

ユニットは最大1つのアイテムを装備でき、通常攻撃型または呪文型として成長します。
シナジーは配置人数に応じて複数段階で強化され、戦士や魔導士などの基本系統に加え、共鳴者、時計仕掛け、寄生虫、錬金術師、星座師などの特殊系統があります。

### アイテム

通常アイテムは8種、合成アイテムは36種です。
合成画面の2枠へ通常アイテムをドラッグすると、組み合わせに応じた1つの合成アイテムを作成できます。

### レリック

通常レリックはラン全体へ継続的な強化を与えます。
古代レリックは幕ボス撃破後などに獲得でき、配置上限、部隊構成、シナジー固有効果といった戦い方そのものを変化させます。

### 記憶の祭壇

ラン終了時に獲得する記憶の欠片で、以下のような恒久要素を解放できます。

- 新規ユニット
- ショップの商品数、アイテムリロール、商品の引き継ぎ
- 戦闘報酬の提示数とリロール
- 古代レリックのリロール、開始時の古代レリック選択
- 初期ゴールド、初期アイテム、配置上限
- 休憩地点などの機能強化

### セーブ

ラン状態、恒久解放、実績、設定はブラウザの `localStorage` に保存されます。
公開版はクライアントだけで動作するため、セーブデータは使用中のブラウザ・端末に紐づきます。

## 開発環境

- Vite 6
- TypeScript 5.6
- 素のDOM + CSS
- 外部ゲームエンジン・UIフレームワーク不使用

### セットアップ

```bash
npm install
npm run dev
```

開発サーバーは [http://localhost:5180](http://localhost:5180) で起動します。

### 本番ビルド

```bash
npm run build
npm run preview
```

生成物は `dist/` に出力されます。

## GitHub Pages

`vite.config.ts` の `base` は、プロジェクトページ
`https://<ユーザー名>.github.io/komalog/`
での公開を想定して `/komalog/` に設定しています。

リポジトリ名や公開先のパスを変更する場合は、`vite.config.ts` の `base` も合わせて変更してください。

## BGMファイル

以下のファイルを `public/assets/` に配置すると、画面に応じて自動再生されます。

- `bgm-title.mp3`
- `bgm-stage1.mp3`
- `bgm-stage2.mp3`
- `bgm-stage3.mp3`

ブラウザの自動再生制限により、初回はクリックなどのユーザー操作後に再生が始まります。

## 主なファイル

- `src/battle.ts` — 10tick/秒の自動戦闘シミュレーション
- `src/screens.ts` — タイトル、マップ、ショップ、報酬、図鑑などの画面UI
- `src/state.ts` — ラン状態、ロースター、星アップ、セーブ処理
- `src/map.ts` — 分岐マップ生成
- `src/meta.ts` — 恒久解放と実績
- `src/settings.ts` — 音量・演出設定
- `src/data/units.ts` — ユニットとシナジー
- `src/data/enemies.ts` — 敵ユニットとフロア別編成
- `src/data/relics.ts` — 通常レリックとアイテム
- `src/data/ancientRelics.ts` — 古代レリック
- `src/style.css` — 全画面のスタイル
